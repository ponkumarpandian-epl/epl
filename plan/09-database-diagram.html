EPL Schema — existing + tournament additions

Visual overview of the full domain schema after the work in 08-general-tournaments.md and 10-team-captain-roster.md lands. The new tournament + bracket subsystem is purely additive; the captain/roster plan adds TeamMember plus a Team.CaptainUserId FK. The Bracket entity is polymorphic — it can attach to either a TournamentCategory (now) or a SeasonGame (future, no schema change needed).

Existing (EPL Season) New (Tournament + Bracket) Shared / polymorphic Identity (ASP.NET) — referenced only

Reading the diagram. Arrows point from the child (FK side) to the parent (PK side). Cyan lines are EPL existing relations. Gold lines are new Tournament-side relations. Green lines are the shared Bracket subsystem. Faint dashed lines are nullable FKs (e.g. Team.CreatedByUserId, TournamentEntry.PlayerXUserId) and future polymorphic edges (SeasonGame → Bracket, Team → BracketParticipant). The small curved dashed loop on Match is the self-reference via NextMatchId — wires winner advancement in single-elimination brackets. Each line is labelled with the FK column name. The polymorphic Bracket parent isn't a DB-level FK — it's a discriminated pair (ParentType, ParentId) enforced in application code; same for BracketParticipant.Source*.

TeamMember

New▶

Per-player roster row. One row per (Team, User). Captain is the one row with IsCaptain = true; players added by the captain are Active immediately (no pending state). A player can self-leave at any time, which flips the row to Removed.

Match

Shared▶

Two-sided match between bracket participants. NextMatchId wires winner advancement for knockouts; null for round-robin. Snapshots the resolved scoring rules at creation time (Round override → Bracket default) so historical scoring stays coherent if admin edits the rules mid-tournament. This snapshot is the authoritative feed to the future scoring feature.

Columns
  • PK · Id (Guid)
  • FK · BracketId · RoundId · NextMatchId (self, nullable)
  • FK · ParticipantAId (nullable) · ParticipantBId (nullable) · WinnerParticipantId (nullable)
  • SlotIndex · Status (MatchStatus enum)
  • ScheduledAt (nullable) · Court (nullable)
  • RulesBestOf · RulesPointsToWin · RulesWinByMargin · RulesPointCap (nullable) — frozen snapshot
  • UNIQUE · (BracketId, RoundId, SlotIndex)
